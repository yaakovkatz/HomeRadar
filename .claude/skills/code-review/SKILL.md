# Code Review Skill

סקירת קוד אוטומטית למערכת HomeRadar - מזהה בעיות, קוד מיותר, ובאגים פוטנציאליים.

## מה לבדוק

### 1. קוד מיותר / לא בשימוש
- משתנים שמוגדרים ולא נקראים
- imports שלא בשימוש
- קבצים שנטענים ולא מנוצלים (כמו locations.json שנטען לתצוגת כרטיסיות בלי סיבה)
- פונקציות שלא נקראות מאף מקום
- קוד מת (אחרי return, בתוך if False, מוערות)

### 2. לוגיקה מיותרת
- סינון/עיבוד שכבר נעשה במקום אחר (כפילות לוגיקה)
- תצוגה שעושה לוגיקת עסקית במקום רק להציג מה שיש ב-DB
- בדיקות מיותרות (בדיקת דבר שתמיד True/False)

### 3. באגים פוטנציאליים
- try/except שבולע שגיאות בשקט (except: pass)
- נתיבי קבצים hardcoded שיכולים להישבר
- encoding issues (חסר encoding='utf-8' בקריאת קבצי עברית)
- race conditions בגישה ל-DB

### 4. עקביות קוד
- סגנון לא אחיד (tabs vs spaces, הזחה)
- שמות משתנים לא עקביים (עברית vs אנגלית)
- הערות שלא תואמות את הקוד

## קבצים לסריקה

סרוק את הקבצים הבאים לפי סדר חשיבות:
1. `main.py` - GUI ותצוגה
2. `database.py` - לוגיקת DB וחילוץ נתונים
3. `ai_agents.py` - סוכני AI
4. `listener.py` - תזמור ראשי
5. `scraper.py` - סריקת פייסבוק
6. `analytics.py` - סטטיסטיקות

## איך להריץ

קרא כל קובץ, זהה בעיות, והצג דוח מסודר:

```
======================================================================
סקירת קוד - HomeRadar
======================================================================

📁 main.py:
  ⚠️ שורה 45: import json לא בשימוש
  ❌ שורה 120: משתנה data מוגדר אבל לא נקרא
  💡 שורה 200: לוגיקת סינון כפולה - כבר נעשה ב-database.py

📁 database.py:
  ✅ לא נמצאו בעיות

סה"כ: X בעיות נמצאו
```

סווג כל ממצא:
- ❌ **בעיה** - צריך לתקן
- ⚠️ **אזהרה** - כדאי לתקן
- 💡 **הצעה** - שיפור אפשרי

## מתי להשתמש

- `/code-review` - סקירה כללית
- `/code-review main.py` - סקירה של קובץ ספציפי
- המשתמש שואל "יש בעיות בקוד?"
- המשתמש שואל "תבדוק את הקוד"
- אחרי שינויים גדולים
